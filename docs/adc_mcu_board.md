## 1. Назначение и параметры

### Назначение
Главная плата гамма-спектрометра на NaI(Tl) с ФЭУ Hamamatsu R1307, выполняющая:
- Аналоговое формирование импульсов с ФЭУ (CR-RC² shaper под NaI)
- Оцифровку формы импульсов
- Цифровую обработку сигналов (DPP)
- Накопление гистограммы (спектра) до **8192 каналов**
- Управление высоковольтным БП через PWM
- Мониторинг ВВ и температуры детектора через АЦП
- Связь с ПК по протоколу shproto / Atom Spectra (BecqMoni-совместимая) через USB
- Хранение спектров и калибровок на внешней SPI-флешке

## 2. Блок-схема платы

```
Делитель ФЭУ+AD8000 → коаксиал U.FL → CR (OPAMP1) → RC (OPAMP2) → PGA ×4 (OPAMP3) → ADC1_IN12
(на плате делителя)   J_sig_in         τ_diff=1мкс   τ_int=510нс    DC=2.1В
                      ±431мВ вход

Параллельно:
  - DAC1_OUT1 (PA4) → R_inj 10к + C_inj 100п → инжекция test pulse
  - TL431DBZ (2.495В) → делители DC ≈0.49В (для OPAMP1/2) и ≈0.528В (для OPAMP3)
  - TIM1_CH1 PWM (PA8) → RC-фильтр → HV_CONTROL
  - ADC2_IN1 (PA0) ← HV_MONITOR
  - ADC4_IN3 (PB12) ← Pt1000 термодатчик (на кристалле)
  - SPI1 (PB3-5, PA15) → W25Q64 флешка
  - USART1 (PA9, PA10) ↔ FT231XS на плате питания
  - SWD (PA13, PA14, NRST)

STM32G474MET6 @ 170 МГц от HSE 16 МГц через PLL (M=4, N=85, R=2) в Boost mode.
```

## 3. Детальное описание узлов

### 3.1. Входной сигнал (от платы делителя)

AD8000 **перенесён на плату делителя** (v1.6). На плату МК приходит уже
усиленный сигнал ±431 мВ (Cs-137) по коаксиалу.

**Разъём J_sig_in** — U.FL (SMD, у края платы).  
Кабель: RG-178, ≤ 100 мм, от платы делителя.

**Входная цепь на плате МК:**
```
J_sig_in (U.FL) ── короткая трасса ── C_diff (100 пФ, CR-каскад OPAMP1)
```
Терминация не нужна: кабель << λ/10 на рабочих частотах < 100 МГц.  
DC = 0 В (AD8000 на делителе работает с ±5В питанием, DC на его выходе ≤ 80 мВ, отсекается C_diff).

**Питание платы делителя (±5V_A):**  
Разъём **J_pwr_div** — JST PH 2.0 мм, 3 пина (+5V_A, AGND, −5V_A).  
±5V_A приходят с USB board через header (пины 1, 2) и транзитом уходят
к делителю. На самой плате МК ±5V_A не потребляются.

| Ref | Компонент | Назначение |
|---|---|---|
| J_sig_in | U.FL SMD | Сигнал от AD8000 на делителе |
| J_pwr_div | JST PH 3-пин | ±5V_A + AGND к делителю |

**Амплитуды на входе шейпера (после U.FL):**
- Am-241 (59.5 кэВ): ±38.5 мВ
- Cs-137 (662 кэВ): ±431 мВ
- 3.5 МэВ: ±2.28 В

*Документация AD8000 (схема, BOM, устойчивость) — см. divider_board.md.*

### 3.2. Test pulse injection

**Назначение:** самотест работоспособности тракта + измерение электронного шума (ENC) для диагностики. **НЕ для абсолютной калибровки** (она делается реальным источником: Cs-137 для энергошкалы, Ra-226 для термокомпенсации — см. `docs/reference_spectra.md`).

**Принцип:** DAC1_OUT1 (PA4) выдаёт ступенчатый импульс через R_inj = 10 кОм и C_inj = 100 пФ на входной узел шейпера (J_sig_in / C_diff, до OPAMP1).

**Номиналы:**
- R_inj = 10 кОм 1% (0402)
- C_inj = 100 пФ NP0 (0402)

**Использование:**
- В покое ЦАП = 1.65 В, DC заблокирован через C_inj
- `TEST_PULSE ‹amp_mV›`: ступенька 1.65 → 1.65+amp В на 10 мкс
- Прошивка: 100-1000 откликов, σ baseline = ENC (**0.31 канала rms** по sim3, см. 3.11.5)
- Команды shproto MODE_TEXT: `TEST_PULSE_ON/OFF/AMPLITUDE`

### 3.3. CR-RC² Shaper на встроенных ОУ STM32G474

**Параметры:**
- **τ_diff = 1 мкс** (CR)
- **τ_int = 510 нс** (RC)
- **Peaking factor = 0.21** для G474 с NaI (τ_d = 230 нс), верифицировано sim2
- Peaking time ≈ 330 нс (G474 sim)
- Длительность импульса ~1.5 мкс
- Undershoot ~7 % от пика
- Шумовая полоса BW_n ≈ 350 кГц

**Обоснование τ_diff = 1 мкс:**

CR-RC² с экспоненциальным входом v_in(t) = A·exp(-t/τ_d):

H(s) = (s·τ_diff) / [(1+s·τ_diff)·(1+s·τ_int)]

Аналитический расчёт по τ_diff (τ_int = 500 нс, τ_d = 230 нс):

| τ_diff | Peak | Undershoot |
|---|---|---|
| **100 нс** | **+0.06** | **−0.09 (БОЛЬШЕ пика!)** |
| 230 нс | +0.20 | −0.31 |
| 500 нс | +0.30 | −0.18 |
| **1000 нс** | **+0.36** | **−0.07** |

Правило: **τ_diff ≥ 3·τ_d** для NaI.

При АЦП 4 Мвыб/с: FWHM импульса ~750 нс = 3 сэмпла, полная длительность ~1.5 мкс = 6 сэмплов. Rise time ~200 нс = 1 сэмпл (на грани, но достаточно для PUR).

**Полезный эффект τ_diff = 1 мкс:** CR с f_low = 160 кГц отрезает 1/f шум встроенных ОУ G474 (доминирует ниже 100 кГц).

#### 3.3.1. CR-дифференциатор на OPAMP1

**Топология:** инвертирующее включение, standalone external mode.

**Распиновка (Table 204 RM0440):**
- VINM = **PA3**
- VINP = **PA1** (DC-смещение)
- VOUT = **PA2**

**Компоненты:**
- **R_cr_in = 10 кОм 1% (0402)** между AD8000_OUT и C_diff — формирует полюс H_CR = −sτ/(1+sτ)
- C_diff = 100 пФ NP0 (0402) между R_cr_in и PA3
- **R_diff = 10 кОм 1% (0402)** между PA2 и PA3
- DC PA1 от TL431DBZ: R_a = 8.2 кОм + R_b = 2 кОм → V_ref ≈ 0.49 В + 100 нФ к GND

**В покое:** V_PA3 = V_PA1 = 0.5 В, V_PA2 = 0.5 В.

#### 3.3.2. RC-интегратор на OPAMP2

**Топология:** инвертирующее включение, standalone external mode.

**Распиновка (Table 204 RM0440):**
- VINM = **PA5**
- VINP = **PA7** (DC, общий с PA1)
- VOUT = **PA6**

**Компоненты:**
- C_couple = 100 нФ X7R (0402) между PA2 и узлом R_int
- R_int = 1 кОм 1% (0402) от C_couple к PA5
- **C_int = 510 пФ NP0 (0402, E96)** между PA6 и PA5
- **R_stab = 1 кОм 1% (0402)** параллельно C_int
- DC PA7 от того же делителя что PA1

**Расчёты:**
- τ_int = R_int × C_int = 1 кОм × 510 пФ = 510 нс ✓
- Полюс H_RC: 1/(R_stab×C_int) = 1/(1к×510п) = 1.96 МГц = 1/τ_int ✓
- DC-усиление: G_DC = R_stab/R_int = 1. Offset OPAMP2 ±3 мВ → ±3 мВ ✓

#### 3.3.3. Топологический комментарий

**AC-связь между каскадами обязательна:** offset встроенных ОУ G474 ±3 мВ накапливается через каскады. AC через 100 нФ (cutoff ~16 Гц) убирает DC.

**Полярность:** AD8000 (−1) × OPAMP1 (−1) × OPAMP2 (−1) × OPAMP3 (+1) = **−1**. Импульсы на ADC отрицательные относительно V_PB1_DC = 2.1 В. В DSP: `amplitude = baseline - sample`.

### 3.4. Выходной буфер на OPAMP3

**Реализация:** PGA mode (internal PGA, без внешних обратных резисторов).

**Распиновка (Table 204 RM0440):**
- VINP = **PB0**
- VINM = **PB2** (внутр. PGA, тест-точка)
- VOUT = **PB1**

**PGA gain в G474:** только +2, +4, +8, +16, +32, +64.

**Конфигурация:** PGA_GAIN = 0b00001 → +4.

**Компоненты:**
- C_buf_in = 100 нФ X7R (0402) от PA6 к PB0
- DC PB0: R_c = 22 кОм + R_d = 5.9 кОм → V_PB0 = 0.529 В
- Развязка: 100 нФ X7R к GND

**DC-уровни:**
- В покое V_PB1 = 4 × 0.529 = 2.116 В ≈ 2.1 В ✓
- Запас от VREF+ 2.5 В: 0.38 В (под undershoot)
- При 3.5 МэВ V_PB1 = 2.116 − 1.83 = 0.29 В ✓

**⚠ OPAMP3 — самый шумящий каскад** (нет аналогового LP-фильтра после, шум до Nyquist 2 МГц). Программная компенсация — LP FIR в DSP (см. 3.11.6).

### 3.5. АЦП

**ADC1 — основной канал:**
- Канал **ADC1_IN12** (internal от OPAMP3_VOUT, физически = PB1)
- VREF+ = 2.495 В от TL431DBZ (U301)
- Sampling 2.5 циклов, total 12.5 циклов
- ADC clock 60 МГц → ~4 Мвыб/с
- Continuous DMA в буфер 4096 слов

**ADC2 — мониторинг ВВ:**
- Канал **ADC2_IN1** (PA0)
- Sampling 247.5 циклов (R_th ≈ 110кОм, требование ≥ 5.8 мкс), триггер 1 кГц, усреднение до 100 Гц

**ADC4 — мониторинг Pt1000 (термокомпенсация):**
- Канал **ADC4_IN3** (PB12)
- Sampling 247.5 циклов
- **Hardware oversampling 256× → эффективные 16 бит** (G474 поддерживает аппаратное усреднение)
- Триггер 10 Гц (один цикл 256 семплов = ~1.5 мс)
- Резолюция: с 16-бит → ~0.02°C на бит при Pt1000 + R303=10к pullup — избыточно для нашей задачи (±0.1°C достаточно)

**ADC3 не используется.**

### 3.7. HV_CONTROL через PWM

**Замена DAC на PWM** (DAC1_OUT2 = PA5 занят OPAMP2_VINM).

**Реализация:**
- TIM1_CH1 на PA8, f_PWM ~100 кГц (ARR = 1700 при HCLK = 170 МГц), ~10 бит
- RC-фильтр после PA8: 2× (R = 10 кОм + C = 100 нФ) → fc = 160 Гц → ripple < 0.1 %
- Далее на шпильку HV_CONTROL → плата ВВ-БП

**Калибровка:** CCR1 ↔ V_HV таблица во флешке.

### 3.8. TL431DBZ и делители DC-смещения

**TL431DBZ** (SOT-23), 2.495 В тип., 50 ppm/°C. Выбран для типизации BOM (TL431 также на HV board U202).

Цепь: +3.3V_A → R301(**330Ом**) → Cathode; REF замкнут на Cathode; C303(100нФ) на REF → GND; Anode → GND. Cathode подключён напрямую проводом к VREF+ STM32 (U302).

**Нагрузки узла 2.495В:** делитель A (0.245 мА) + делитель B (0.089 мА) + **Pt1000 pull-up R303 (0.225 мА, ратиометрия)** + VREF+ ADC (~0.2 мА). При R301=330Ом ток входа 2.44 мА → на TL431 остаётся ≥1.6 мА > 1 мА min ✓ (при старых 470Ом было на грани — изменено по аудиту 2026-06-11).

**Делители (E96, 1 %):**

A — для OPAMP1+OPAMP2 (V_ref ≈ 0.49 В на PA1+PA7):
- R312 = 8.2 кОм + R313 = 2.0 кОм → V_A = 2.495 × 2.0/(8.2+2.0) = 0.490 В ✓
- Развязка C321 100 нФ X7R к GND

B — для OPAMP3 (V_PB0 ≈ 0.528 В, PGA×4 → V_PB1 = 2.113 В):
- R310 = 22 кОм + R311 = 5.9 кОм → V_B = 2.495 × 5.9/(22+5.9) = 0.528 В ✓
- Развязка C320 100 нФ X7R к GND

**Стабильность:** дрейф 50 ppm/°C × ΔT. При ΔT = 30°C сдвиг 0.15 % — пренебрежимо (NaI дрейф ~3 % за 30°C доминирует).

### 3.11. Бюджет электронного шума и ENC

#### 3.11.1. Источники шума на входе AD8000

Параллельное R_in 50 Ом ‖ R_g 100 Ом = 33 Ом:
- e_R = √(4kT × 33) = **0.74 нВ/√Гц**

R_fb 1.5 кОм, приведённый ко входу:
- e_R_fb = √(4kT × 1500) / 15 = **0.33 нВ/√Гц**

AD8000 datasheet:
- Voltage noise: **1.6 нВ/√Гц** (>100 кГц)
- Current noise i_n- × (R_g ‖ R_fb) = 3.0 пА/√Гц × 93.75 Ом = **0.28 нВ/√Гц**

Суммарный:
- e_in² = 0.74² + 1.6² + 0.28² + 0.33² = 3.30
- **e_in = 1.82 нВ/√Гц** — доминирует AD8000

#### 3.11.2. Шум встроенных ОУ STM32G474

Datasheet DS12288 Table 80:
- @ 1 кГц: 250 нВ/√Гц (1/f)
- @ 10 кГц: 90 нВ/√Гц

White floor (выше corner ~100 кГц): **50 нВ/√Гц** (по Figure 34).

**τ_diff = 1 мкс даёт f_low = 160 кГц у CR-каскада** → 1/f шум ниже 160 кГц отрезается.

#### 3.11.3. Шумовая полоса CR-RC²

BW_n ≈ π/(8τ_int) × (1 + 1/(1+τ_int/τ_diff)) ≈ **350 кГц**.

#### 3.11.4. σ_baseline по каскадам (приведено к выходу OPAMP3)

*Аналитический расчёт ниже — оценочный, выполнен для PGA×2 (v1.4). Авторитетный результат: sim3 (94.4 мкВ rms, см. 3.11.5).*

| Каскад | σ (оценка, PGA×2) |
|---|---|
| AD8000 (gain тракта ×10.8) | 1.82 × √350кГц × 10.8 = **11.6 мкВ rms** |
| OPAMP1 (gain далее ×2) | 50 × √350кГц × 2 = **59 мкВ rms** |
| OPAMP2 (gain далее ×2) | 50 × √350кГц × 2 = **59 мкВ rms** |
| OPAMP3 (до Nyquist 2 МГц, peak averaging) | √(35² + 81²) = **88 мкВ rms** |

С программным LP fc=500 кГц: σ_OPAMP3 → 60 мкВ rms.

#### 3.11.5. Итоговый бюджет

**Без программного LP:**
- σ_total² = 11.6² + 59² + 59² + 88² = 14841
- **σ_total = 122 мкВ rms**
- В каналах: 122/610 мкВ = **0.20 канала rms (4096)** = **0.40 канала rms (8192)** = **0.94 канала FWHM**

**С программным LP fc=500 кГц:**
- σ_total² = 10545
- **σ_total = 103 мкВ rms = 0.34 канала rms (8192)**

**Электронный вклад в FWHM на Cs-137 (канал 1138, G474 sim3):**
- σ_total = **94.4 мкВ rms** (sim3 LTspice), 0.31 канала rms (8192)
- FWHM_noise = 0.73 канала
- Электронный вклад в FWHM: 0.73/1138 × 100 % = **0.064 %** (vs расч. 0.085 %)

**Полный бюджет FWHM системы:**

| Источник | Вклад FWHM |
|---|---|
| Статистика фотоэлектронов R1307 + NaI (паспорт) | 6.3 % |
| HV-ripple типичный | 0.3 – 0.5 % |
| Нелинейность АЦП 12 бит (ENOB ~10.5) | 0.1 % |
| Электроника | 0.085 % |
| Тепловой дрейф ±30 °C | 0.05 % |

**FWHM_total = √(6.3² + 0.5² + 0.1² + 0.085² + 0.05²) = 6.32 %**

#### 3.11.6. Выводы

1. **Электроника не лимитирует разрешение** (0.085 % vs 6.3 % ФЭУ).
2. **Целевая FWHM 6.0-6.5 % достижима** при хорошем монтаже модуля.
3. **Главный «вор» FWHM после ФЭУ — HV-ripple.** Вопрос платы ВВ-БП.
4. **OPAMP3 — самый шумящий каскад.** Программная компенсация: FIR 8-tap @ fc=500 кГц.
5. **σ_baseline на test pulse = 0.31 канала rms** (sim3). Если больше — диагностика разводки.
6. **5σ детект. порог = ~0.6 кэВ** — огромный запас.

#### 3.11.7. Caveats

- White floor G474 = 50 нВ/√Гц — оценочное. Если 80 нВ/√Гц → σ ×1.6 = 195 мкВ rms = 0.64 канала FWHM. Всё равно << ФЭУ.
- Не учтены наводки от цифровых линий, кварца, PWM. Качественный layout критичен.

## 8. Прошивка — общая структура

### 8.1. Архитектурное решение

**Подключение к BecqMoni:** через **shproto-протокол по UART** (FT231XQ → USB-C). НЕ как аудиоустройство.

**Распределение работы:**
- **MCU** делает: peak detection, baseline, pile-up rejection, накопление гистограммы 8192 каналов, **термокомпенсацию**.
- **BecqMoni** делает: отображение, fitting пиков, экспорт XML, идентификация изотопов.

**Главное ТЗ прошивки** — **термокомпенсированный** спектр на выходе. Положение пика Cs-137 должно оставаться на том же канале при ΔT ±30°C. Полная процедура термокалибровки описана в `reference_spectra.md` (раздел "Архитектура прошивки и термокомпенсация").

### 8.2. Структура main loop

```c

int main(void) {

// thermal_calibration.c
// Главный модуль — без него прибор не годен для серьёзной спектроскопии

typedef struct {
    float alpha;       // линейный коэф дрейфа [1/°C]
    float beta;        // квадратичный коэф [1/°C²]
    float T_ref;       // опорная температура калибровки [°C]
    uint32_t cal_date; // timestamp последней калибровки
} ThermalCal_t;

static ThermalCal_t thermal_cal;
static volatile float current_gain_corr = 1.0f;  // обновляется в UpdateTemperature

float ReadTemperature(void) {
    // Pt1000 (700-102BAA-B00), R_pullup=10k → +3.3V_A
    // ADC4 с hardware oversampling 256x → эффективные 16 бит
    uint32_t adc_oversampled = ADC4_Read();        // 16-bit value (12-bit native × 256 average)
    // Делитель: V_pt = 3.3V × R_pt / (10000 + R_pt)
    // ADC = 65535 × V_pt / 3.3V (16-bit)
    // → R_pt = 10000 × adc / (65535 - adc)
    float r_pt = 10000.0f * adc_oversampled / (65535.0f - adc_oversampled);

    // Callendar-Van Dusen (для T > 0°C):
    //   R(T) = R0 × (1 + A*T + B*T²),  A=3.9083e-3,  B=-5.775e-7
    // Решая для T (квадратичное уравнение, для нашего диапазона −50..+100):
    //   T = (-A + sqrt(A² - 4B × (1 - R/R0))) / (2B)
    const float R0 = 1000.0f;  // 1кΩ @ 0°C для Pt1000
    const float A = 3.9083e-3f;
    const float B = -5.775e-7f;
    float ratio = r_pt / R0;
    float disc = A*A - 4.0f*B*(1.0f - ratio);
    float t_celsius = (-A + sqrtf(disc)) / (2.0f * B);
    return t_celsius;
}

void UpdateTemperature(void) {
    static uint32_t last = 0;
    if (HAL_GetTick() - last < 100) return;  // 10 Гц
    last = HAL_GetTick();

    float T = ReadTemperature();
    float dT = T - thermal_cal.T_ref;
    // gain ушёл — компенсируем обратной величиной
    current_gain_corr = 1.0f / (1.0f + thermal_cal.alpha*dT + thermal_cal.beta*dT*dT);
}

// DSP_processing.c
// ВНИМАНИЕ: полярность импульсов отрицательная относительно V_PB1_DC = 2.1В
//          (нечётное число инверсий через тракт). amplitude = baseline - sample.

void ProcessPulse(uint16_t* samples, int count) {
    int baseline = ComputeBaseline(samples);
    int peak_idx, peak_val;
    FindPeakNegative(samples, count, &peak_idx, &peak_val);

    if (IsPileUp(samples, peak_idx)) {
        pile_up_count++;
        return;
    }

    int amplitude_raw = baseline - peak_val;
    // === Температурная коррекция (главное!) ===
    int amplitude = (int)(amplitude_raw * current_gain_corr);

    int channel = ApplyEnergyCalibration(amplitude);

    if (channel >= 0 && channel < 8192) {
        spectrum[channel]++;
        total_count++;
    }
}

// peak_tracker.c
// Периодически отслеживает положение известного фонового пика (K-40 1461 кэВ)
// Если ушёл — автоматически корректирует gain offset

void PeakTracker_Update(void) {
    static uint32_t last = 0;
    if (HAL_GetTick() - last < 60000) return;  // раз в минуту

    int k40_ch = FitPeakInRange(spectrum, K40_CH_EXPECTED-100, K40_CH_EXPECTED+100);
    if (k40_ch > 0) {
        float drift_pct = 100.0f * (k40_ch - K40_CH_EXPECTED) / K40_CH_EXPECTED;
        if (fabsf(drift_pct) > 1.0f) {
            // дрейф > 1% → авто-коррекция
            thermal_cal.T_ref += drift_pct / thermal_cal.alpha / 100.0f;
            Flash_SaveThermalCal(&thermal_cal);
        }
        if (fabsf(drift_pct) > 5.0f) {
            shproto_send_text("WARNING: gain drift > 5%, recalibrate!");
        }
    }
}
```

## 9. Совместимость с BecqMoni — протокол shproto

UART 600 кбит/с → FT231XS → USB-C. Совместим с BecqMoni / nanopro.

### Структура пакета

```
0xFF [START 0xFE] [CMD] [PAYLOAD with byte-stuffing] [CRC16 LSB] [CRC16 MSB] [FINISH 0xA5]
```

**Escape:** 0xFE, 0xFD, 0xA5 → `0xFD <инверсия>`.
**CRC-16:** Modbus, init 0xFFFF.

### Режимы

| Код | Режим | Направление |
|---|---|---|
| 0x01 | MODE_HISTOGRAM | МК → ПК |
| 0x02 | MODE_PULSE | МК → ПК |
| 0x03 | MODE_TEXT | оба |
| 0x04 | MODE_STAT | МК → ПК |
| 0xF3 | MODE_BOOTLOADER | оба |

### Текстовые команды

`-inf`, `-sta`, `-sto`, `-rst`, `-cal`, `-nos <число>`.

### Buffer

8192 каналов × 4 байта = 32768 байт. Возможен downsampling до 8192 байт при передаче.

### Реализация на МК

Парсер с https://github.com/Am6er/nanopro/blob/master/shproto/__init__.py портируется на C 1-в-1.

`-inf` ответ (40 строк, 5 × double calibration + CRC32 + serial) — снять с реального устройства или из исходников эталонной прошивки.

## 12. Открытые аналитические вопросы

### 12.1. ✅ Бюджет шума и ENC (РЕШЕНО sim3)
См. 3.11. **σ_sim3 = 94.4 мкВ rms = 0.31 канала rms** (sim3 LTspice). Расч. аналит. (без LP): 122 мкВ / 0.40 канала. Электроника **0.064 %** FWHM << ФЭУ 6.3 %.

### 12.2. ✅ Стабильность AD8000 при ×15 (РЕШЕНО v1.4)
Схема AD8000 перенесена на плату делителя (v1.6) — см. divider_board.md и sim1_ad8000.cir.
R_fb = 1.5 кОм при ×15 устойчив. Полоса 150 МГц. C_fb = 1 пФ обязателен для компенсации C_par.

### 12.3. Pile-up при 300 кимп/с (ОТКРЫТО, не блокер PCB)
P = 1 − e^(−R·τ) ≈ 45 % при R = 300 кимп/с и τ = 1.5 мкс.

Решается в прошивке/симуляции:
- Эффективность rise-time PUR при 1 сэмпле на rise time
- Effective live time
- При критичности — τ_int 250 нс, FWHM 6.5-7 %, pile-up 23 %

### 12.4. ✅ Тепловой дрейф (РЕШЕНО, в норме)
±30 °C даёт сдвиг калибровки 0.15 % (TL431DBZ, 50 ppm/°C) — << дрейф NaI ~3 % за 30°C.

### 12.5. Ringing коаксиала (не блокер)
RG-178 50 Ом + 50 Ом терминатор → согласовано. При длине < 30 см проблем нет.

## 14. Roadmap (v2 и далее)

Текущая v1.6 — компромисс **минимальная стоимость / разрешение 6.0-6.5 % FWHM на NaI(Tl) R1307 Ø63×63**.

**Важный вывод из 3.11:** улучшать электронику бессмысленно (на два порядка ниже ФЭУ). Путь к лучшей FWHM — через детектор.

### 14.1. Дискретные ОУ вместо встроенных G474
Offset 150 мкВ vs 3 мВ → меньше DC-дрейфа. OPA838, OPA320, MAX44267. В v1 заложить footprint-pad'ы.

### 14.2. CSA вместо voltage amp
Анод → виртуальная земля FET-input ОУ + R_f 100 МОм + C_f 1 пФ. Имеет смысл только при ФЭУ с FWHM 2-4 %.

### 14.3. Быстрый АЦП для LaBr3
4 Мвыб/с не хватает для τ_decay 25 нс. STM32H7A3 или внешний AD9235.

### 14.4. QSPI на флешку
Нет в G474. Есть в H7/L4+. Для v1 не критично.

### 14.6. Стратегия

**v1.6 (текущая):** бюджетный спектрометр, FWHM 6.0-6.5 %, ~22 €. Все симуляции выполнены. Следующий этап: KiCad разводка.

**v2 (если хочется лучше):** **смена детектора, не электроники**. R6231 → 4.5 % FWHM.

**v3 (энтузиасты):** LaBr3 + H7 + быстрый АЦП. FWHM 3-4 %. ~150 €.
