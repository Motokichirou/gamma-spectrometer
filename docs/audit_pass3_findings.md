## #3 ИСПРАВЛЕНО — divider_board.md: C411↔C413 и C412↔C414 перепутаны

**Где:** `docs/divider_board.md`, таблица "Компоненты AD8000"

**Верификация нетлистом:**
- Net `+5V_A`: C413 pin1, C414 pin1, U401 pin7(+VS) → C413/C414 — bypass **+5V_A** к AGND
- Net `−5V_A`: C411 pin2, C412 pin2, U401 pin4(-VS) → C411/C412 — bypass **−5V_A** к AGND
- C411/C412 pin1 → GND (между GND и −5V_A → отрицательная шина bypass) ✓

**Было:**
```
| C411 | 4.7 мкФ X5R | +5V_A развязка | 0805 |
| C412 | 100 нФ X7R  | +5V_A развязка | 0402 |
| C413 | 4.7 мкФ X5R | −5V_A развязка | 0805 |
| C414 | 100 нФ X7R  | −5V_A развязка | 0402 |
```

**Стало:**
```
| C411 | 4.7 мкФ X5R | −5V_A развязка | 0805 |
| C412 | 100 нФ X7R  | −5V_A развязка | 0402 |
| C413 | 4.7 мкФ X5R | +5V_A развязка | 0805 |
| C414 | 100 нФ X7R  | +5V_A развязка | 0402 |
```

**Причина:** Во время рисования схемы конденсаторы расставлялись в другом порядке, чем записаны в таблице. Схема правильная — обе шины имеют bypass. Документация неверно именовала компоненты.

## #4 ИСПРАВЛЕНО — adc_mcu_board.md Section 3.5: ADC2 sampling — 24.5 вместо 247.5

**Где:** `docs/adc_mcu_board.md`, раздел 3.5 "АЦП"

**Было:**

- Канал **ADC2_IN1** (PA0)
- Sampling 24.5 циклов, total 12.5 циклов
- Триггер 1 кГц, усреднение до 100 Гц
```

**Стало:**
```
**ADC2 — мониторинг ВВ:**
- Канал **ADC2_IN1** (PA0)
- Sampling 247.5 циклов (R_th ≈ 110кОм, требование ≥ 5.8 мкс)

**Причина:**
- `hv_board_schematic.md` Section 7 явно рассчитывает: R_th ≈ 110кОм → требуется Sampling ≥ 247.5 цикла при ADC clock 42.5 МГц (5.8 мкс). ADC4 (NTC) в том же разделе 3.5 правильно указан как 247.5 cycles.
- "24.5" = очевидная опечатка (пропущена "7"). "total 12.5 циклов" — ошибочно скопировано из ADC1.

## #5 ИСПРАВЛЕНО — usb_power_board.md: неполный бюджет +5V_A

**Где:** `docs/usb_power_board.md`, таблица "Бюджет тока из USB"

**Верификация нетлистом:** Net `+5V_A` содержит U104 pin2 (IN TPS60400) → TPS60400 питается от +5V_A, а не напрямую от +5V_USB. При I_out = 25 мА (−5V_A для AD8000) charge pump потребляет ≈28 мА из +5V_A.

**Было:**
```
| +5V_A | AD8000 (на плате делителя) | 30 мА |
| -5V_A | AD8000 (на плате делителя) | 25 мА |
| **Итого из USB** | | **~370 мА** ✓ (< 500 мА USB 2.0) |
```

**Стало:**
```
| +5V_A | AD8000 (на плате делителя) + TPS60400 вход (charge pump) | ~58 мА |
| −5V_A | генерируется TPS60400 из +5V_A; нагрузка AD8000 ~25 мА | — |
| **Итого из USB** | | **~373 мА** ✓ (< 500 мА USB 2.0) |
```

**Примечание:** суммарный ток из USB остаётся в норме (~373 мА). Но строка `-5V_A 25мА` вводила в заблуждение — она учитывалась как отдельная USB-нагрузка, тогда как этот ток уже включён в +5V_A через TPS60400.

## Дополнительные наблюдения (не ошибки, но стоит знать)

### TPS60400 переключательный шум на +5V_A (250 кГц)
TPS60400 (250 кГц) питается от +5V_A — той же шины что и AD8000 (+VS). Местный bypass C110/C111 уменьшает шум, но при разводке PCB важно держать TPS60400 и его LC-фильтр подальше от зоны AD8000. Альтернатива — рассмотреть питание TPS60400 непосредственно от +5V_USB (до L101), но это потребует изменения схемы.

### Старые логические имена в теле hv_board_schematic.md
Тело документа (разделы 2–7) использует логические имена R12, C24, R16, R17, R23–R47, R28–R37 и т.д. — эти имена не совпадают с KiCad-рефами 2xx-серии из BOM (раздел 10). Это не ошибка — документ намеренно использует короткие имена в теле с маппингом только в BOM. Аналогично сделано в usb_power_board_schematic.md. Для PCB-разводки авторитетный источник — нетлист.
