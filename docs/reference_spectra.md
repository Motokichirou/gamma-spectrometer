## Выводы для нашего проекта (NaI Ø63×63 + R1307)

### Целевой бюджет резолюции
- На основе sim3: σ_electronics = 94.4 мкВ rms = 0.064% FWHM @ Cs-137
- Доминирует PMT/crystal статистика: ~6-6.5% (паспорт R1307 + NaI 63×63)
- **Расчётная FWHM @ Cs-137: 6.0-6.5%** (между 5% NaI 40×40 и 7.25% CsI 80×80)

### Что должна обеспечить наша прошивка
1. **Многоточечная калибровка** — полином 3-4 порядка хранится во flash. Минимум 3 точки.
2. **Раздельная low-E коррекция** — ниже 100 кэВ NaI(Tl) нелинейный, нужен либо отдельный полином, либо больше точек.
3. **Pile-up rejection** — при 300 cps ожидаемый pile-up без PUR = 40%, с PUR (rise-time discrim, τ=200 нс) = 8-10%. При 30k cps PUR обязателен.
4. **Sum-peak обработка** — coincidence summing работает на больших объёмах, нужен анализ временного окна.
5. **Совместимый XML-экспорт** — BecqMoni XML формат: `<EnergySpectrum>` с polynomial calibration + `<DataPoint>` × 8192.
6. **Длительные накопления** — пример Ra: 11 часов, 40k секунд, 27M counts. Прошивка должна стабильно работать сутками.
7. **🔴 Peak tracking + gain drift detection** — реальный пример из Th-файла: между калибровкой и измерением gain ушёл на 9%. Прошивка должна:
   - Отслеживать известный референсный пик (например K-40 в фоне @ 1461 кэВ — всегда присутствует)
   - Автокорректировать gain (через HV или цифровой множитель) если дрейф > 1%
   - Сохранять историю калибровок с timestamp во flash
   - Выводить warning через shproto если дрейф > 2% от последней калибровки
8. **High-E запас по диапазону** — Tl-208 2614 кэВ должен быть на ch ~4500/8192 (55%), не выше 80%, чтобы не было обрезания Compton continuum выше пика.

### 🔥 Архитектура прошивки и термокомпенсация (главное ТЗ)

**Принцип:** Наш прибор подключается к BecqMoni через **shproto-протокол по UART** (не как аудиоустройство). MCU делает peak detection и накапливает гистограмму. BecqMoni только отображает и экспортирует данные.

**Задача прошивки:** отдавать BecqMoni термокомпенсированный спектр. То есть положение пика Cs-137 на канале N должно оставаться на канале N при любой температуре окружающей среды.

#### Источники температурного дрейфа

| Источник | Коэффициент | На ±30°C |
|---|---|---|
| Светоотдача NaI(Tl) | −0.3%/°C | ±9% |
| Усиление R1307 (PMT) | −0.5%/°C типично | ±15% |
| HV-БП TL431 опора (U202) | 50 ppm/°C × 9% YOUTAI margin | ~±0.15-0.45% |
| MCU ADC опора TL431 (U301) | 50 ppm/°C × 9% YOUTAI margin | ~±0.15-0.45% |
| Электроника тракта (G474 OPAMP, AD8000) | <0.01%/°C | ±0.03% |
| **Итого без компенсации** | | **~24% дрейф канала Cs-137** |
| **После компенсации** | | **<0.5% (цель)** |

Без термокомпенсации спектрометр непригоден для серьёзной спектроскопии — при работе на улице зимой/летом канал Cs-137 будет уезжать на сотни каналов.

#### Процедура термокалибровки (one-time, в лаборатории)

1. **Подготовка:**
   - Поместить прибор в термокамеру (или быстро меняющую T среду — например пенопластовый бокс с льдом → постепенное нагревание комнатным воздухом)
   - Поместить рядом источник **Ra-226** (даёт ~10 пиков от 186 до 2204 кэВ — позволяет валидировать линейность не только в одной точке)
   - Подключить прибор по shproto к ПК с BecqMoni

2. **Прогон:**
   - Прошивка переходит в режим "thermal calibration" (через команду shproto MODE_TEXT, например `-tcal_start`)
   - Каждую минуту: запись (timestamp, T_NTC, спектр 8192) во flash циклический буфер
   - Параллельно T меняется от −10°C до +50°C за ~6 часов (~10°C/час)
   - Объём данных: 360 минут × (4 байт NTC + 32 КБ спектр) ≈ 12 МБ — флешка W25Q64 = 8 МБ. **Решение:** сохранять не полный спектр, а только положения известных пиков (Bi-214 609 кэВ и Bi-214 1764 кэВ — 8 байт × 360 записей = 3 КБ, влезает).

3. **Post-processing:**
   - Скрипт на PC (Python) читает журнал из flash
   - Для каждого пика строит зависимость channel(T)
   - Фитит полином 2-го порядка: ch_at_T(T) = ch_at_T0 × (1 + α(T−T0) + β(T−T0)²)
   - Сохраняет коэффициенты (α, β, T_ref) обратно во flash через shproto команду

4. **Рабочий режим:**
   - Прошивка читает T_NTC каждую секунду (медленный ADC4)
   - Вычисляет `gain_corr(T) = 1 / (1 + α·ΔT + β·ΔT²)` где ΔT = T_current − T_ref
   - Применяет к каждому импульсу: `channel = round((baseline − sample) × gain_corr(T))`
   - Двойной механизм:
     - **Цифровая коррекция** — основной, быстрый, без задержек
     - **HV авто-подстройка** — опционально для больших отклонений (ΔT > 15°C), медленнее (~1 сек установления), но физически правильнее (PMT работает в номинальной рабочей точке)

5. **Periodic re-calibration:**
   - Прошивка отслеживает положение K-40 (1461 кэВ — всегда присутствует в фоне)
   - Если дрейф > 1% от калиброванного положения → автокоррекция gain offset
   - Если дрейф > 5% → warning в shproto, требовать перекалибровки источником

#### Что хранится в flash

```
[0x000000] Заголовок (magic, version, serial, T_ref, last_cal_date)
[0x000100] Энергетическая калибровка: polynomial coefs (3-4 порядка) × float
[0x000200] Термокалибровка: α, β, T_ref + history последних 5 калибровок
[0x001000] Журнал температуры: кольцевой буфер (T, timestamp, ch_K40)
[0x010000] Сохранённые спектры (опционально, для long-term archive)
```

#### Точность Pt1000 + ADC4

**Датчик:** Pt1000 (Honeywell 700-102BAA-B00 / артикул 32208572) — платиновый, корпус M222, leaded.
- R = 1000 Ω @ 0°C
- Чувствительность: +3.85 Ω/°C (положительный TCR)
- Точность: ±0.15% (class A) → ±0.075°C @ 50°C
- Долговременная стабильность: ±0.05% / 10 лет (типично для Pt)

**Схема:** делитель +3.3V_A → R303 (10 кΩ pullup) → Pt1000 → GND.
- При 25°C: V_ADC = 3.3 × 1097/(10000+1097) = **0.326В**
- Self-heating: I = 0.297 мА → P_Pt = 0.097 мВт → нагрев < 0.05°C ✓

**ADC4 hardware oversampling 256× → 16 эффективных бит:**
- 65536 counts × 19.86 мкВ/count
- Чувствительность 1.03 мВ/°C → **~52 counts/°C → резолюция 0.02°C на бит** (избыточно)

**Формула (Callendar-Van Dusen для Pt1000, обращённая):**
```c
R_pt = R_pullup × adc / (65535 - adc);          // R_pullup = 10000 Ω, adc 16-bit
ratio = R_pt / 1000;
disc = A² - 4B × (1 - ratio);                   // A=3.9083e-3, B=-5.775e-7
T_celsius = (-A + sqrt(disc)) / (2B);
```

Для нашего диапазона −10..+50°C ошибка от формулы Callendar-Van Dusen относительно линейной (T = (R−1000)/3.85) составляет <0.05°C — можно использовать линейку без потери точности. Полная формула на случай работы при −40..+85°C.

**Бюджет ошибки термоизмерения:**
- Pt1000 паспорт: ±0.075°C
- Self-heating: ±0.05°C
- ADC4 oversampling шум: <0.02°C
- **Итого: ±0.1°C** → дрейф gain ошибка <0.1°C × 0.8%/°C = 0.08% → канал Cs-137 ошибка <1 канала.

Это **на 5× лучше** чем NTC + Steinhart, и **долговременно стабильнее** на годы.
