# Шпаргалка: типовые ошибки и правила проверки схем

> **Подгружать при каждой работе с KiCad схемами.**

---

## Правила проверки скриншотов

- Проходить по КАЖДОМУ пину явно: "пин X соединён с Y — совпадает с документацией?"
- Не писать "правильно" до завершения полного обхода всех узлов
- Сверять с даташитом, не только с документацией проекта
- При сомнении — просить пользователя прислать крупный план узла

## Ошибки, допущенные в проекте

### 1. Полярность конденсаторов на отрицательной шине

**Правило:** На отрицательной шине (−5V_A, −HV и т.п.) полярные конденсаторы подключаются **НАОБОРОТ**:
- `+` пластина → GND
- `−` пластина → отрицательная шина

**Прецедент:** C112 (TPS60400 выход), C113 (−5V_A фильтр) на USB/Power board — полярность обратная: `+` к GND.

**Проверять:** Любой полярный конденсатор на шинах с отрицательным потенциалом.

---

### 2. BAS516 — снаббер первичной цепи, НЕ выпрямитель вторичной

**Правило:** В flyback на MAX1847+ATB322524:
- BAS516 (**D201**) стоит в **первичной** цепи: Анод → Drain Q201 / T201 pin4
- Катод → снаббер (C208 10нФ ‖ 2×BZT52C18 последовательно → GND)
- На вторичной обмотке (T201 pin3) **нет отдельного диода** — выпрямление выполняют BAV23S внутри CW умножителя

**Прецедент:** Изначально ошибочно планировался BAS516 как выпрямитель вторичной → схему пришлось перерисовывать.

---

### 3. Полярность вторичной обмотки ATB322524

**Правило:** По даташиту ATB322524-0110-T000:
- Pin 1 = начало первичной (dot), Pin 4 = конец первичной
- Pin 2 = начало вторичной (dot), Pin 3 = конец вторичной

**Подключение:**
```
T201 pin 1 → +5V_HV
T201 pin 4 → Drain Q201 (IRLML0040)
T201 pin 2 (dot) → GND
T201 pin 3 → C209 1нФ (первый конденсатор CW)
```

**Прецедент:** pin2 и pin3 перепутаны местами → неверная полярность выхода → перерисовка.

---

### 4. CS пин MAX1847 — Source до резистора, не после

**Правило:** Q201 Source подключён к CS (pin 13) И к R205 (0.1Ом). CS снимает напряжение на узле Source ДО резистора R205:

```
Q201 Source ─┬─ CS (pin 13 MAX1847)
             ├─ R205 0.1Ом ─── GND
             └─ C205 100пФ ─── GND   (параллельно R205)
```

---

### 5. SHDN MAX1847 — активный HIGH, прямое подключение от MCU

**Правило:** HV **выключено** по умолчанию (безопасное состояние):
- MCU HV_ENABLE = HIGH (3.3В) → SHDN = HIGH → MAX1847 работает → HV включён
- MCU HV_ENABLE = LOW / floating → R203 (100кОм) тянет SHDN к GND → MAX1847 стоп

P-MOSFET AO3415 **не используется** — убран. Схема: MCU → R204(10кОм) → SHDN; R203(100кОм) → GND.
V_IH(SHDN) ≈ 0.7×VL ≈ 3В; MCU 3.3В > 3В ✓

---

### 6. BAV23S в CW умножителе — используется как ОДИН диод

**Правило:** BAV23S содержит два диода последовательно (pin1=A1, pin3=средняя точка, pin2=K2).
В CW умножителе используется как **единый диод повышенного напряжения**:
- **pin1 (A)** → анод
- **pin2 (K)** → катод
- **pin3 → NC (не подключён, висит в воздухе)**

Поставить `X` (No Connect) на pin3 каждого BAV23S в KiCad во избежание ERC ошибки.

**Зачем:** 2 диода последовательно = 2×250В = **500В** обратное напряжение.
В нашем CW на ступень приходится 2×94.6В = 189В << 500В ✓

## Работа с файлами нетлиста KiCad — правила верификации

### Принцип: только .net, report.txt удалён

**`.net` файл** (`gamma_spectrometer.net`) — экспортируется из текущей схемы и всегда актуален. Единственный авторитетный источник для верификации.

**report.txt удалён** (был лог аннотации — показывал значения на момент перенумерации, не текущее состояние; вводил в заблуждение).

### Как работать с .net файлом экономно (без чтения 4000+ строк)

Файл делится на два раздела:
1. `(components ...)` — определения компонентов с ref и value
2. `(nets ...)` — список цепей с узлами

**Шаг 1 — найти цепь по имени:**
```
Grep: "AMP_OUT"  →  строка с (net (code "18") (name "AMP_OUT"))
```

**Шаг 2 — прочитать только эту цепь (Read с offset):**
```
Read offset=3645 limit=10  →  видим все узлы сети AMP_OUT
```

**Шаг 3 — проверить значение компонента:**
```
Grep: "comp.*R305"  →  строка с (comp (ref "R305"))
Read offset=<строка> limit=5  →  видим (value "10k")
```

**Никогда не читать весь .net файл подряд** — это тысячи токенов впустую.
**Никогда не читать .kicad_sch файлы** — ещё больше, сжигают токены без пользы.

| C208 (снаббер) | 10нФ | Демпфирование звона на Drain |
| D202+D203 (снаббер) | 2×BZT52C18 последовательно = 36В | Ограничение Vdrain; запас 40−36 = 4В |
| C209–C216 (CW конд.) | 1нФ C0G 500В 0805, CC0805JKNPOBBN102 | Рабочее напряжение на ступень ~189В << 500В |
