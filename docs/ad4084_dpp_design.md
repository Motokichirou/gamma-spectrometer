# AD4084 — анализ для DPP-версии спектрометра

> Дата: 2026-06-10. Источник: `docs/refs/ad4084.pdf` (Rev 0, 07/2025, 94 стр.) — прочитан детально.
> **РЕШЕНИЕ ПРИНЯТО (2026-06-10): делаем v1 (аналоговый CR-RC² + G474 ADC). v2 на AD4084 — апгрейд на будущее.**
> Этот документ — готовое ТЗ для MCU board v2.0, когда придёт время: AD4084 обрастёт errata, ценой и наличием.
> Совместимость: v2 = замена ТОЛЬКО MCU board в стеке, межплатные разъёмы те же; Divider/USB/HV board не трогаются (на Divider добавится 1 резистор R_inj для сдвига baseline).

---

## 1. Ключевые факты из даташита

### Общее
- 16-bit SAR, 1.25–20 MSPS, SINAD 92.5 dB @ 1 кГц, SNR 92.4 dB @ 1 МГц
- Транзишн-шум 0.52 LSB rms; INL ±6 ppm; BW входа 272 МГц
- Потребление 66.8 мВт @ 20 MSPS (динамическое)
- Корпус: **BGA-49, 5×5 мм, шаг 0.65 мм** ⚠ (горячий воздух / заводская сборка)
- Spectroscopy явно указана в Applications

### Питание (3-4 домена!)
- **VDD33 = 3.3В** (аналоговый) — от нашей +3.3V_A
- **VDD11 = 1.1В** (ядро), **IOVDD = 1.1В** (интерфейс)
- Вариант А: внутренние LDO — подать **VDDLDO = 1.4–2.7В**, VDD11/IOVDD генерируются сами. ⚠ Внешняя нагрузка на внутренние LDO **запрещена** (нельзя запитать от них трансляторы).
- Вариант Б: внешний 1.1В LDO на VDD11 + IOVDD (VDDLDO не подключать) — **наш выбор**, т.к. от этой же шины питаются уровневые трансляторы.
- Все развязочные конденсаторы **встроены в корпус** (470нФ VDD33, 1.88мкФ VDD11, 220нФ IOVDD/VDDLDO).
- Sequencing: VDD11/IOVDD → VDD33 → цифровые входы → REFIN. После сброса 300 мкс до доступа.

### 🔴 Критическое ограничение №1: уровни интерфейса 1.1В
- Все data-interface пины (DCLK, DCS, SDOB, GPIO0-3) — домен IOVDD (1.045–1.26В максимум!)
- Config SPI входы (CS, SCLK, SDI, CNV) толерантны до 2.5В — **но НЕ 3.3В**
- Выходы качаются 0–IOVDD (1.1В) — STM32 на 3.3В их не прочитает (VIH ≈ 1.87В)
- **Вывод: нужны двунаправленные уровневые трансляторы 3.3В↔1.1В** (например 2× TXU0104/SN74AXC4T245) + LDO 1.1В

### 🔴 Критическое ограничение №2: SPI = только FIFO
- Стр. 67: *"Continuous access to filtered data results is available only through the LVDS data interface. SPI data interface access to filtered results is only made via the FIFO."*
- Непрерывный поток 20 MSPS доступен **только по LVDS** (FPGA/H7-территория)
- **С STM32G474 единственный режим — событийный захват пачками через FIFO** (что нам и нужно)

### Reference
- REFIN требует **внешний прецизионный 3.0В источник** (допуск ±5 мВ!): LTC6655-3.0 (лучший, 2 ppm/°C), **ADR4530B** (0.02%, 2 ppm/°C, дешевле) или LT6657
- Внутри — буфер с 9.4 мкФ резервуаром (буферный усилитель не нужен)
- Это **заменяет и превосходит** наш TL431 (50 ppm/°C → 2 ppm/°C, ×25 лучше температурная стабильность шкалы!)

### Вход
- Полностью дифференциальный: FS = ±V_REFIN = ±3В дифф. (6 Vpp), LSB = 91.55 мкВ
- CM = 1.5В задаётся выходом **CMO** (V_REFIN/2, импеданс 700Ом, фильтровать RC)
- Easy Drive: ток входа 5 мкА/MSPS линеаризованный → драйвер маломощный достаточен
- Анти-алиас фильтр (Table 8, 20 MSPS/16бит): **R_FILTIN=25Ом, C_FILTIN=100пФ** (дифф.), AUXIN: R=100Ом, C=25пФ
- AUXIN+/− — вспомогательные входы линеаризации, подключаются к тому же сигналу через 4×R
- Рекомендуемые драйверы (Table 12): **ADA4945-1** (FDA, 4мА, 1.8нВ/√Гц, 145МГц — lowest power) ← наш выбор

### Event Detection (стр. 53-55) — аппаратный триггер импульсов!
- HI_THRESHOLD/LO_THRESHOLD (регистры 0x21-0x24) сравниваются с каждым сэмплом
- Гистерезис настраиваемый (0x1F)
- Латентность: **t_EVT = 2–3 цикла CNV** (100–150 нс @ 20 MSPS) от пересечения порога до ALERT
- ALERT/HI_DTCT можно вывести на GPIO → EXTI на MCU
- Событие может **аппаратно запускать FIFO**

### Result FIFO (16384 × 16 бит, стр. 55-62)
| FIFO_MODE | Режим | Применение |
|---|---|---|
| 00 | выключен | — |
| 01 | Immediate trigger | захват N=WATERMARK сразу (для baseline-замеров) |
| **10** | **Event trigger, post-event** | **захват WATERMARK сэмплов ПОСЛЕ события — наш основной режим** |
| 11 | Event trigger, read all 16K | пред+пост история, но читать все 16384 (6 мс — слишком медленно) |

- Статусы FIFO_FULL / FIFO_READ_DONE → на GPIO
- После захвата FIFO надо перевооружить (запись по config SPI: MODE=00 → MODE=10)
- Чтение: header + M×байт по data SPI (DCS/DCLK/SDOB), DCLK до 50 МГц, 1 или 4 линии

### Цифровые фильтры (Sinc1/Sinc5/Sinc5+comp, decimation 2–1024)
- Работают на полном потоке, выход можно подавать на event detection и FIFO
- ⚠ С FIFO требуют FILTER_SYNC (GPIO) перед каждым захватом — усложнение
- **Решение: фильтр НЕ используем** (FILTER_SEL=00), сырые 20 MSPS сэмплы в FIFO, фильтрация в MCU на пачке

### Коррекции в чипе
- OFFSET регистр: ±2048 LSB (±11.7 мВ)
- GAIN регистр: ±1.56% FS, шаг 0.003% — для подстройки тракта, **недостаточен для термокомпенсации PMT** (нужно до ±24%) → термокомпенсация остаётся в MCU (математика) и/или через HV

## 2. Предлагаемая архитектура (G474 + AD4084, Figure 51 из даташита)

```
PMT анод → AD8000 ×(−15) [Divider board — БЕЗ ИЗМЕНЕНИЙ] → коаксиал U.FL →
[MCU board v2.0]:
  ADA4945-1 (FDA, SE→diff, gain ≈1.3, VOCM=CMO 1.5В, питание +5V_A)
  → анти-алиас 25Ом/100пФ дифф. (+ AUXIN цепь 100Ом/25пФ)
  → AD4084 @ ~17 MSPS (CNV от TIM STM32 через транслятор)
  → Event Detection (порог чуть выше baseline) → GPIO → EXTI G474
  → FIFO mode 10, WATERMARK = 48 (2.8 мкс окно)
  → G474 читает пачку по SPI+DMA (42.5 МГц) → DSP → гистограмма 8192 → shproto
```

### 2.1. Драйвер: нужен ли ADA4945 при живом AD8000?

AD8000 (1.5 ГГц CFA на делителе) способен раскачать входы AD4084 напрямую — Easy Drive входы (5 мкА/MSPS линеаризованные) и анти-алиас 25Ом/100пФ для него лёгкая нагрузка.

**Препятствие — полярность/диапазон:** AD8000 даёт импульсы В МИНУС (baseline 0В → −2.28В @ 3.5 МэВ), а вход AD4084 допускает только −0.1…+3.1В.

**Вариант А — прямое подключение (без FDA):**
- Сдвиг baseline: инжекция тока в инвертирующий вход AD8000 от −5V_A через R_inj ≈ 2.7к → baseline = +2.8В, импульсы вниз до +0.52В ✓ (**+1 резистор на Divider board**)
- IN+ ← сигнал; IN− ← DC от CMO (RC-фильтрованный); AUXIN± аналогично через 100Ом
- Event detection по **LO_THRESHOLD** (импульсы вниз); прошивка: `amplitude = baseline − min` (как в v1)
- Минус: single-ended = половина дифф. диапазона → **15 эфф. бит**. Но шум тракта ~110 мкВ ≈ 1.2 LSB — квантование не лимитирует, потеря бита ничего не стоит
- Шум −5V_A через R_inj: вклад ≈0.25 нВ/√Гц на вход — пренебрежимо (LC-фильтр 81 дБ уже стоит)

**Вариант Б — ADA4945-1 FDA на MCU board:**
- Полный диапазон (16 бит), инверсия в плюс, CM от CMO по канону ADI, изоляция кикбэка от коаксиала
- +$4, +1 корпус

**Решение для v2-платы: развести footprint ADA4945 с обходом 0Ω перемычками.** Прототип собирать по варианту А; FDA — запасной вариант без перерезки платы.

### Расчёт мёртвого времени (FIFO mode 10)
- Окно 48 сэмплов × 2 байта ≈ 100 байт @ 42.5 Мбит/с ≈ 19 мкс чтение
- Перевооружение: config SPI запись ~1 мкс + 2-3 CNV
- **Итого ~20-25 мкс/импульс**: 1 kcps → 2.5% dead time ✓; 10 kcps → 25% ⚠; 30 kcps → не работает ✗
- Для пищевого/экологического мониторинга (≤1 kcps) — отлично

### Захват пика при латентности триггера 100-150 нс
- Анти-алиас RC + замедление фронта: порог на ~15% амплитуды → пик через 300-400 нс после пересечения → захват стартует до пика ✓
- Baseline: отдельные периодические immediate-захваты (mode 01) между импульсами + цифровое слежение

### Порог обнаружения
- Шум тракта на входе AD4084 (10 МГц полоса): ~110 мкВ rms → 5σ ≈ 0.6 мВ ≈ **<1 кэВ** даже на сырых сэмплах ✓

**Добавляется:**

|---|---|---|
| AD4084BBCZ (BGA-49) | АЦП 16-bit 20 MSPS | $40-60? (Rev 0, цена не устоялась) |
| ADA4945-1CPZ (LFCSP-16) | FDA драйвер SE→diff | $4 |
| ADR4530BRZ (SOIC-8) | 3.0В reference, 2 ppm/°C | $4 |

| Анти-алиас: 25Ом×2, 100Ом×2, 100пФ, 25пФ×2, RC на CMO | пассивы | $0.5 |

**Пины G474 освобождаются:** PA1-PA7, PB0-PB2 (OPAMP/DAC цепи) → используются под второй SPI (data) + GPIO (ALERT/FIFO_FULL/FIFO_READ_DONE EXTI) + TIM CNV.

**PCB:** 4 слоя сохраняются (LVDS не нужен — SPI). BGA 0.65мм dogbone: JLCPCB 4-layer справляется (0.2мм сверло / 3.5mil трасса), но желательна заводская сборка этого узла.

### Прошивка
- + Драйвер AD4084 (config SPI: ~15 регистров init)
- + FSM: ARM → EXTI(event) → DMA read FIFO → DSP → re-arm
- + DSP на пачке 48 сэмплов: baseline-вычитание, цифровой фильтр/фит пика, PUR по форме
- Гистограмма/shproto/термокомпенсация — без изменений (16-битная амплитуда >> 3 → 8192 каналов)
- Симуляции: sim2 (CR-RC²) → не нужна; вместо неё Python-моделирование цифрового фильтра; sim3 пересчитать на новый тракт

## 4. Что даёт и чего НЕ даёт

### Даёт
1. **Шкала стабильнее ×25**: ADR4530 2 ppm/°C vs TL431 50 ppm/°C (термодрейф электроники практически исчезает; остаётся только PMT+кристалл)
2. **16 бит честных**: Am-241 59.5 кэВ ≈ 820 LSB (vs 51 LSB на 12-бит) → можно расширить диапазон вниз до рентгеновской области 10-20 кэВ (Np L-линии, Pb флуоресценция)
3. **Полная форма импульса 20 MSPS** → PSD (n/γ дискриминация с CLYC/стильбеном в будущем), rise-time PUR
4. **Software-defined shaping** → LaBr3 (τ=25нс) без переделки железа
5. Минус 15 пассивов аналогового шейпера и их разброс/дрейф

### НЕ даёт
1. **FWHM на NaI+R1307 НЕ улучшится**: 6.3% статистики ФЭУ доминируют (наша электроника уже 0.064%)
2. **Высокий CPS НЕ получится через SPI**: мёртвое время чтения FIFO ограничивает ~5-10 kcps практических (vs 30 kcps у аналогового тракта с G474 ADC)  ← парадокс: DPP-версия на SPI **хуже по скорости счёта**, чем текущий план!
3. Цена +$50-70, BGA-сборка, Rev 0 риски

---

## 5. Риски
1. **Rev 0 (07/2025)** — errata неизвестны, цена не устоялась, наличие у дистрибьюторов под вопросом
2. **BGA-49 0.65мм** — самостоятельная пайка сложная; план: JLCPCB assembly узла АЦП
3. Сложность инициализации (94 стр., ~15 регистров, sequencing питания)
4. Транслятор уровней в цепи CNV — джиттер (для нашей полосы сигнала ≤1.5 МГц не критичен)

## 6. Открытое решение

| | v1: аналоговый CR-RC² (текущий план) | v2: DPP на AD4084 |
|---|---|---|
| Готовность схем | 100% (ERC ✓, BOM ✓) | MCU board redesign ~2-4 недели |
| FWHM @ Cs-137 | 6.0-6.5% | 6.0-6.5% (то же) |
| Макс CPS | ~30 kcps | ~5-10 kcps (SPI-лимит) |
| Порог | ~1 кэВ | <1 кэВ, рентген 10-20 кэВ возможен |
| Темп. стабильность шкалы | TL431 50 ppm/°C + компенсация | ADR4530 2 ppm/°C + компенсация |
| Гибкость (LaBr3, PSD) | нет | да |
| Цена | ~22€ | ~75-90€ |
| Сборка | вся рукой | BGA = заводская сборка узла |
| Риск | минимальный | Rev 0 + BGA |

**Рекомендация Claude:** v1 довести до железа (схемы готовы!), v2 проектировать параллельно как отдельную MCU board v2.0 — стек плат позволяет заменить ОДНУ плату, не трогая USB/HV/Divider. Обе версии MCU board совместимы по межплатным разъёмам.
