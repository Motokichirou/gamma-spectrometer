# Прошивка

<!-- README-I18N:START -->
**Русский** | [English](../en/firmware.md)
<!-- README-I18N:END -->

Прошивка STM32G474 снимает сформированные импульсы с АЦП, детектирует и измеряет
каждый импульс, накапливает гистограмму на 8192 канала и стримит её на хост по
[протоколу shproto](protocol.md), чтобы [BecqMoni](https://github.com/Am6er/BecqMoni)
отображал спектр.

## Переносимое ядро

Код обработки сигнала лежит в `firmware/core/` и **не зависит от HAL** — компилируется
где угодно и общий для bring-up платы Nucleo и боевой MCU board:

| Модуль | Ответственность |
|---|---|
| `shproto.c/.h` | Парсер/билдер кадров: CRC16-MODBUS, байт-стаффинг, FSM |
| `spectrum.c/.h` | Гистограмма `uint32` на 8192 канала |
| `dsp.c/.h` | Детектор импульсов: трекер baseline, отбраковка наложений, квадратичный LSQ-фит вершины |
| `pulsegen.c/.h` | Генератор тестовых импульсов (источник встроенного самотеста) |
| `selftest.c/.h` | FSM самотеста ENC / линейности |
| `app.c/.h` | Обработка команд (`-sta/-sto/-rst/-stt/-mode/-inf/-cal/-rcal/-wcal/-calclr/-thr/-tst`) и спектр + статус 1 Гц |

Ядро общается с железом через несколько порт-функций (`app_port_uart_send`,
`app_port_millis`, …), реализованных в платоспецифичном клее.

**Энергокалибровка** хранится во внутренней flash (`cal_store.c`, последняя страница
2 КБ, Bank2 page 127 dual-bank = `0x0807F800`, magic + CRC16) и **переживает ребут**.
Хранятся 11 «сырых» слов формата BecqMoni; протокол `-cal`/CRC-32 описан в
[protocol.md](protocol.md). Запись возможна из BecqMoni (кнопка «Записать в устройство»)
или нашим тулом (`-wcal`); чтение — `-cal` (формат BecqMoni) или `-rcal` (наш формат).
На этапе 1 (без flash) хуки калибровки — пустые заглушки.

## Этапы разработки

| Этап | Железо | Цель | Статус |
|---|---|---|---|
| 1 | Nucleo-G474RE | shproto через VCP → BecqMoni рисует тестовый спектр | ✅ работает |
| 2 | Nucleo-G474RE | петля DAC→ADC: реальные импульсы, peak detect, реальная гистограмма | ✅ работает |
| 3 | Боевая MCU board | OPAMP-шейпер, реальные импульсы ФЭУ, ВВ, термокомпенсация Pt1000 | ⏳ ждёт железа |

На Nucleo этап 2 замыкает петлю целиком на кристалле: пуассоновский планировщик
на TIM7 стреляет DAC реалистичными формами импульсов, перемычка заводит DAC (PA4/A2)
обратно на АЦП (PA0/A0) на 2.83 Мвыб/с, и тот же DSP, что поедет на боевую плату,
строит гистограмму. Измеренная пропускная способность ≈ 6000 имп/с принятых из ≈ 8500
сгенерированных.

## DSP импульсов (`dsp.c`)

Для каждого отсчёта АЦП детектор:

1. **Трекает baseline** через EMA, обновляющийся только в коридоре `±threshold`,
   так что ни импульсы, ни провалы не утаскивают baseline.
2. **Триггерит** при превышении порога отклонением, с гистерезисом: триггер
   взводится заново только после спада ниже `threshold/2`.
3. **Отбраковывает наложения**: как только развитый пик (`≥ 2×threshold`) начинает
   спадать, повторный рост от минимума спада помечает событие как pile-up и отбрасывает.
4. **Измеряет амплитуду** квадратичным LSQ-фитом (парабола) по `2m+1` отсчётам вокруг
   максимума, беря высоту вершины — несмещённая суб-LSB оценка, прямо отображаемая в
   канал гистограммы на 8192 бина.

Полярность настраиваема (`+1` для петли Nucleo, `−1` для боевой платы, где импульсы
идут ниже baseline). Горячий путь форсируется в `-O2` даже в Debug-сборках: на
2.83 Мвыб/с неоптимизированный ISR не успевает за скоростью полубуфера.

## Подсистема самотеста

Тестовый генератор работает и как встроенный самотест, управляется командой `-tst`:

| Команда | Действие |
|---|---|
| `-tst off` | Генератор выключен (по умолчанию при старте) |
| `-tst spec` | Воспроизводит реальное распределение амплитуд Cs-137 (выборка по inverse-CDF) |
| `-tst mono <amp> [t_us]` | Моноэнергетические импульсы с фиксированной амплитудой/периодом |
| `-tst stat` | Текущие среднее / σ / FWHM детектированных амплитуд |
| `-tst enc` | Автотест ENC + линейности (5 амплитуд × 1000 событий) |

Автотест `-tst enc` возвращает один текстовый кадр с центроидом, σ и FWHM для каждой
амплитуды, плюс наклон прямой и INL. Характерный прогон на железе:
`k = 2.0048 ch/code`, `INL max = 0.025 %FS`, `FWHM @ Cs-эквивалент = 0.66 %` —
подтверждает на реальном кремнии, что вклад электроники + DSP пренебрежим рядом с
≈6.3 % статистики ФЭУ (см. [simulations.md](simulations.md)).

> Генератор **выключен при старте** и работает только при явном включении. На боевой
> плате вход АЦП — реальный сигнал ФЭУ, поэтому генератор не должен запускаться сам.

## Сборка и прошивка (headless)

```bash
# Сборка (IDE не нужна; путь -import использует backslash)
"…/stm32cubeidec.exe" --launcher.suppressErrors -nosplash \
  -application org.eclipse.cdt.managedbuilder.core.headlessbuild \
  -data /tmp/cube_hb_ws -import 'C:\gamma-spectrometer\firmware\gamma_stage1' \
  -build gamma_stage1/Debug

# Прошивка по SWD
STM32_Programmer_CLI.exe -c port=SWD mode=UR reset=HWrst \
  -w ".../Debug/gamma_stage1.elf" -v -rst
```

`printf` из newlib-nano требует линкер-флаг `-u _printf_float` для отчётов самотеста
с плавающей точкой.

Пошаговая настройка — в [`firmware/README.md`](../../firmware/README.md).
