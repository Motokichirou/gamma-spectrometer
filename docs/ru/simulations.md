# Симуляции и бюджет шумов

<!-- README-I18N:START -->
**Русский** | [English](../en/simulations.md)
<!-- README-I18N:END -->

Аналоговый тракт проверялся в LTspice до разводки. Все три симуляции выполнены;
исходные деки и Python-скрипты анализа лежат в `simulation/`.

| Sim | Дек | Скрипт | Предмет |
|---|---|---|---|
| 1 | `sim1_ad8000.cir` | `analyze_sim1.py` | Устойчивость и полоса входного каскада AD8000 |
| 2 | `sim2_shaper.cir` | `analyze_sim2.py` | Отклик шейпера CR-RC² и энергетическая калибровка |
| 3 | `sim3_noise.cir` | `analyze_sim3.py` | Полный бюджет шумов электроники |

## Sim 1 — входной каскад AD8000

С `Rfb = 1.5 кОм`, `Cfb = 1 пФ`, `Cpar = 1 пФ`:

- полоса ≈ **99 МГц**, peaking < 0.5 дБ.
- **`Cfb = 1 пФ` обязателен** на PCB делителя для устойчивого, хорошо демпфированного отклика.

## Sim 2 — шейпер CR-RC²

PGA ×4, делитель DC-смещения даёт baseline `V_out_DC = 2.13 В`.
Пик-фактор с поправкой на баллистический дефицит `PF = 0.202` (GBW OPAMP G474
13 МГц чуть снижает идеальные 0.209; прежние «0.36» — ошибка аналитического расчёта).

Итоговое отображение энергия → канал (из 8192):

| Источник | Энергия | Канал |
|---|---|---|
| Am-241 | 59.5 кэВ | 101 |
| Cs-137 | 662 кэВ | 1138 |
| Co-60 | 1.33 МэВ | 2277 |
| Tl-208 | 2.6 МэВ | 4470 |
| Потолок | 3.5 МэВ | 6021 |

## Sim 3 — шум

Суммарный приведённый ко входу шум электроники:

- `σ_total = 94.4 мкВ rms`
- `σ_baseline = 0.31 канала rms`

### Бюджет шумов

| Вклад | FWHM |
|---|---|
| Статистика ФЭУ R1307 + NaI(Tl) | 6.3 % |
| Пульсации ВВ (типично) | 0.3–0.5 % |
| Нелинейность АЦП (12-бит, ENOB ~10.5) | 0.1 % |
| Шум электроники (σ = 94.4 мкВ rms) | **0.064 %** |
| **Итого** | **~6.32 %** |

Шум электроники **на два порядка ниже** статистики ФЭУ, так что энергетическое
разрешение ограничено ФЭУ, а не электроникой. Позже это подтвердилось на железе:
самотест ENC прошивки намерял FWHM @ Cs-эквивалент = 0.66 % от одного только цифрового
тракта (см. [firmware.md](firmware.md)).
