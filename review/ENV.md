# ENV.md — окружение, инструменты, скиллы (независимое ревью)

Дата: 2026-06-15. Объект: гамма-спектрометр NaI(Tl) Ø63×63 + ФЭУ Hamamatsu R1307.
Режим: read-only по проекту, все артефакты ревью — только в `review/`.

## Скиллы
Проверен доступный набор скиллов (code-review, verify, run, find-skills, simplify,
update-config, security-review и др.). **Доменно-релевантных** (анализ аналоговой
электроники / SPICE / проверка физических расчётов) среди них нет — это скиллы
под код-ревью ПО и конфигурацию харнесса. Поэтому верификация выполнена напрямую
инструментами (LTspice + Python/numpy/scipy + собственные нетлисты с `.meas`),
без подмены доменной процедуры неподходящим скиллом. Трекинг — через встроенные Task-*.

## Формат выхлопа LTspice (как «курил» проект и как курил я)
- Проектные `analyze_sim*.py` парсят бинарный `.raw` вручную: заголовок UTF-16LE до
  маркера `Binary:`, далее transient — time = 8 байт double, переменные = 4 байта float
  (noise — все 4 байта). Воспроизведено: sim2 PF=0.202, sim3 σ=94.4 мкВ — совпали.
- Я предпочитал **`.meas` → текстовый `.log`** (потенциалы динодов, пики, амплитуды);
  полный `.raw` читал пакетом `ltspice` только там, где нужна форма (sh_pulse).

## Дерево (что инвентаризировано)
- `pcb/gamma_spectrometer.net` — нетлист KiCad 9 (5875 строк) — **первоисточник связей**.
- `docs/*.md` — описания плат (divider/adc_mcu/hv/usb), cheatsheet, session_context.
- `simulation/` — sim1 (AD8000), sim2 (shaper), sim3 (noise): `.cir` + `.raw/.log` +

  но документированный пик-детект прошивки разобран — см. REVIEW §сэмплирование).

## Созданные артефакты (review/sim/)
| Файл | Что делает |
|---|---|
| `sh_gbw.cir/.log` | шейпер: PF при GBW ОУ 13МГц vs 2ГГц (проверка «G474 душит форму») |
| `sh_ballistic.cir/.log` | баллистический дефицit: пик при фикс. заряде vs τ_scint |
| `sh_pulse.cir/.raw` | один импульс Cs-137, мелкий шаг — форма/undershoot/семплирование |
| `divider_load.cir/.log` | АКТИВНЫЙ делитель (Q401-403): потенциалы динодов k=0 vs 30кимп/с |
| `divider_passive.cir/.log` | ПАССИВНЫЙ делитель — для сравнения просадки усиления |
| `detector_calcs.py` | физика детектора, токи делителя, усиление под нагрузкой, pile-up, баллистика |
| `check_sampling.py` | форма импульса + ошибка пик-детекта (max/парабола/сплайн) |
| `check_aliasing.py` | спектр импульса + восстанавливаемость при 4 Мвыб/с |

Запуск: `& "C:\Program Files\ADI\LTspice\LTspice.exe" -b review/sim/<файл>.cir`,
затем `python review/sim/detector_calcs.py` (и `check_*.py`).
