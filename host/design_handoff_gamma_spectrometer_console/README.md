# Передача в разработку: Пульт гамма-спектрометра (ГаммаПульт)

> Диагностический и калибровочный пульт самодельного гамма-спектрометра.
> NaI(Tl) + ФЭУ Hamamatsu R1307, МК STM32G474, USB-CDC (виртуальный COM, 600000 бод),
> бинарный протокол **shproto**, гистограмма спектра на **8192 канала**.

---

## 1. Обзор

Настольное Windows-приложение — инструмент инженера для ввода прибора в строй и диагностики:
выбор COM-порта и подключение, запуск набора, живой спектр и телеметрия, прогон самотеста ENC,
чтение лога кадров протокола, отправка текстовых команд (`-sta` / `-sto` / `-rst` / `-inf` / `-cal` / `-tst`).
Эстетика — лабораторно-приборная: высокая плотность информации, графики в центре внимания,
тёмная тема, опора на компоновку/цвет/отступы/типографику, а не на тени и скругления.

## 2. О файлах этого пакета

Файлы в бандле — **дизайн-референсы, сделанные в HTML/CSS** (статичная дизайн-спека на «канве»).
Это **не production-код для копирования**. Задача — **воссоздать эти макеты в целевой среде проекта**.

**Целевая среда (из ТЗ): C++ / Dear ImGui + ImPlot.**
- Одно ресайзящееся окно, докуемые панели (`ImGui` docking branch / `DockSpace`), тёмная тема.
- Виджеты — стандартные ImGui: `Button`, `Combo`, `InputText`, `BeginTable`, `BeginTabBar`, `Checkbox`.
- Графики — ImPlot: `PlotLine` / `PlotStairs` / `PlotShaded`, лог-ось (`ImPlotScale_Log10`), курсор/`DragLineX`, `Annotation`.
- HTML здесь нужен только чтобы зафиксировать **раскладку, палитру, размеры, типографику и поведение состояний**.
  Раздел 9 даёт прямую трансляцию токенов в `ImGuiStyle` / `ImPlotStyle`.

Если позже понадобится веб-обёртка — JSX-файлы из бандла переиспользуемы как есть; но первичная цель — ImGui.

## 3. Уровень проработки

**Высокий (hi-fi).** Финальные цвета (hex), типографика, размеры строк/панелей, сетка отступов,
состояния всех ключевых компонентов и 5 состояний интерфейса заданы точно. Воссоздавать пиксель-в-пиксель
средствами ImGui/ImPlot, сверяясь с разделами 6–9.

## 5. Состояния интерфейса (5)

Все состояния влияют на индикатор связи, доступность кнопок и акценты.
**Красный — строго опасность (ВВ) и ошибки. Один акцент (синий) — активные состояния/набор.**

| Состояние | Индикатор связи | Кнопка действия | Телеметрия | Спектр | Самотест |
|---|---|---|---|---|---|
| **Отключено** | `idle` серый ● «Нет связи · порт закрыт» | синяя **Подключить** + «⟳ Порты» | нули/прочерки, плоский спарклайн | пустой график + CTA «→ Подключить · затем `-sta`» | заблокирован, плейсхолдер |
| **Идёт подключение** | `accent` ● пульс «Подключение… запрос -inf» | серая «Подключение…» (disabled) + «Отмена» | нули | пусто | заблокирован |
| **Подключено · простой** | `ok` зелёный ● «Связь установлена · простой» | красная **Отключить** + `-rst` | нули, «Набор остановлен» | пусто/готов | доступен (кнопки активны) |
| **Идёт набор** | `ok` зелёный ● «Связь · набор» | красная **Отключить** + `-rst` | cps крупно (акцент) + спарклайн, drop %, Σ, ● пульс «ИДЁТ НАБОР» | живая гистограмма, курсор | таблица + графики + k/INL |
| **Ошибка связи** | `danger` ● мигание «Ошибка связи · таймаут кадра» | синяя **Переподключить** + `-rst` | замороженные последние значения (dim) + красный баннер «RX timeout 1500 мс · CRC err 3» | замороженный/пусто | заблокирован |

Поля идентификации (`Серийник`, `Прошивка`, `Сборка`, `Калибр. k`) заполняются из ответа `-inf` / `-cal`;
в «Отключено»/«Подключение» — прочерки `—`.

## 6. Панели — подробно

### 6.1 Подключение (300×auto)
- Ряд «COM-порт»: метка 64px + `Combo` на всю ширину (`COM7 — STM32 Virtual ComPort`).
  В подключённых/подключающихся состояниях combo **disabled**.
- Ряд «Скорость»: read-only поле `600000`, справа caption `бод · 8N1`.
- Ряд кнопок (зависит от состояния, см. §5). Высота кнопок 26px.
- Разделитель → **строка статуса связи**: точка-индикатор (9px, с гало) + текст цветом состояния.
- Разделитель → блок идентификации (метка 78px + моно-значение): Серийник `SN-04F2A1`,
  Прошивка `shproto 2.3.1`, Сборка `2025-11-04`, Калибр. k `2.0473 кан/код` (значение акцентом).
- В «Ошибка»: внизу красный баннер с деталями.

### 6.2 Статус / Телеметрия (300×flex)
- «Время набора» — моно 13–18px, `00:12:47`.
- «Скорость счёта» — **крупно**: моно 38px / 600, акцентный цвет в наборе (`1 842 cps`).
  Под числом — **спарклайн** cps во времени (SVG-полилиния; в ImPlot — мини `PlotLine` без осей).
- «Отбраковано» — значение (warn-цвет) + полоса-meter (заливка warn-цветом, доля %).
- «Суммарный счёт» — `1 412 933`.
- Строка набора: ● пульс + «ИДЁТ НАБОР» (акцент) / «Набор остановлен» (dim) / «Набор прерван» (danger).

### 6.3 Спектр (доминанта, 1fr×flex) — активная панель подсвечена снизу шапки акцентом
- Шапка: заголовок «СПЕКТР», справа meta `8192 кан` и **сегмент-переключатель Лин/Лог**
  (активная половина — сплошной акцент, белый текст).
- Полоса статистики над графиком (моно 11px, nowrap): `Σ 1 412 933` · `пик 1352` · `⌖ 1352 · 2607 · 662 кэв`.
- График: гистограмма/линия по 8192 каналам, лог-ось Y (тики 1 / 10 / 100 / 1k / 10k),
  ось X — каналы (0 / 2048 / 4096 / 6144 / 8192). Линия `--trace #6fb0f0`, лёгкая заливка под ней.
- **Курсор**: вертикальная пунктирная янтарная линия (`--cursor #e0a23c`) + чип-отсчёт
  «канал N / счёт M» рядом. Подписи фотопиков `Cs-137 · 662`, `K-40 · 1460`.
- Модель спектра (для тестовых данных) — см. `plots.jsx → spectrumCounts()`:
  низкоэнергетическая «стена» + комптоновский континуум + пики Pb-Kα, обратное рассеяние,
  Cs-137 (662 кэв, канал ~1352), K-40 (1460 кэв, канал ~2990). В проде — реальная гистограмма из STATUS-кадров.

### 6.4 Самотест · ENC (340×flex)
- Кнопки: **▶ Прогон ENC** (primary при подключении) + `Combo` «Тест-режим: меандр».
- Таблица результатов (`BeginTable`, моноширинная, табличные цифры), столбцы:
  **Ампл. код | Центроид | σ | FWHM(%)**. Выбранная строка — подсветка акцентом;
  строка вне допуска — warn-цвет. Пример строк: 500→1024/18.2/9.8% … 3800→7779/28.1/6.0%.
- Строка вывода: `k = 2.0473 кан/код` · `INL = 0.18 %FS` (ok-цвет) · `R² = 0.9998`.
- Два мини-графика: **Линейность** (центроид vs амплитуда — точки + пунктирная подгонка) и
  **FWHM** vs амплитуда (спадающая кривая, warn-цвет).

### 6.5 Консоль · shproto (1fr×190)
- Лог кадров (моно 11.5px, `--plot-bg` фон): таймстемп (faint) + направление + текст + hex.
  Цвета направления: **TX** акцент, **RX** ok-зелёный, **ERR** danger-красный, **sys/··** dim.
- Чипы быстрых команд: `-sta -sto -rst -inf -cal -tst` (акцентные), `-help` (приглушённый).
- Строка ввода: поле с приглашением `›` и мигающим кареткой-акцентом + кнопка **Отправить** (primary при связи).

### 6.6 Дорожная карта (заглушки)
Пункты меню **«Высокое напряжение»** (0…−1500 В) и **«Термокалибровка»** — disabled, с иконкой замка.
Заложить место/вид, не реализовывать. При активации ВВ — обязательна **явная красная индикация опасного напряжения**.

## 8. Взаимодействия и поведение
- **Подключить/Отключить** переключают весь стейт окна (§5). Подключение → авто `-inf` и `-cal` для идентификации.
- **Лин/Лог** — мгновенная смена шкалы Y графика спектра (в ImPlot — `ImPlotScale_Log10` ↔ `_Linear`).
- **Курсор спектра** следует за мышью по оси каналов, показывая канал/счёт (и энергию по `k`).
- **Прогон ENC** заполняет таблицу самотеста и оба мини-графика; считает `k`, `INL`, `R²`.
- **Чип команды** подставляет текст в строку ввода; **Отправить**/Enter шлёт кадр, ответ — в лог.
- **Анимации сдержанные**: пульс индикатора набора (~1.15 c), мигание тревоги (~1 c), каретка ввода.
  Никаких декоративных бесконечных анимаций на контенте.
- Реальная отрисовка спектра — по приходу STATUS-кадров (cps, t, drop) и блоков гистограммы.

## 10. Протокол / модель данных (контекст)
- Транспорт: USB-CDC, виртуальный COM, **600000 бод, 8N1**. Протокол **shproto** (бинарные кадры с CRC).
- Команды (текстовые, шлются в кадре): `-sta` старт набора, `-sto` стоп, `-rst` сброс,
  `-inf` инфо (серийник/прошивка), `-cal` калибровка (`k`, `INL`), `-tst` тест-режимы/ENC.
- Статус-кадры: время набора `t`, скорость счёта `cps`, доля отбракованных `drop`, счётчик CRC-ошибок.
- Гистограмма: **8192 канала**. Калибровка энергии: `E ≈ k · канал` (пример `k = 2.0473`).
- Самотест ENC: линейность `k` (кан/код), интегральная нелинейность `INL` (%FS), `FWHM` по нескольким амплитудам.
- В макете значения **синтетические/правдоподобные** — заменить реальными из прибора.

## 11. Ассеты
Иконографика — инлайновый SVG (стрелки, замок, кнопки окна, иконки пустых состояний). Внешних растров нет.
Шрифт `IBM Plex Mono` — Google Fonts (в проде: подключить `.ttf` через `ImGui::AddFontFromFileTTF`,
плюс системный Segoe UI для пропорционального). Поддержка кириллицы: грузить glyph range
`GetGlyphRangesCyrillic()`.
